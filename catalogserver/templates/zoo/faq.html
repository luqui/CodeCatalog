FAQ

What is this place?

CodeCatalog is a lot like a wiki for source code. We aim to socially build a database of high-quality, well-documented, instantly-usable code in a variety of languages.

So it's like GitHub/SourceForge/Google Code?

Not exactly. CodeCatalog focuses on small, reusable snippets of code. So this isn't the place for your RPG game or complete website—although many of the generally applicable functions and classes you used to create it do belong here. For example: API integration, functions for working with geometry, networking utilities, natural language processing tools, ...

How do I use code from the site?

Just copy and paste, or use our tools. Copy and paste and duplicate to your heart's content: we tag the snippets from the site so that you can use our tools to scan your code for updates and bugfixes. That means (most of) the usual evils of copying and pasting are mitigated.

What are these // CodeCatalog Snippet comments?

Those are the tags that our tools use to cross-reference the code in your codebase with the code on the site. The URL brings you to the page with the documentation for that snippet. It's okay to change the code between the tags; if you run the updater tool, it will ask you if you want to upload your changes or keep them for yourself.

How do I put new code on the site?

First, you need to be logged in to change anything on the site except for filing bugs. Once you are logged in, search for it. If you find a spec (a collection of snippets that do the same thing, possibly in multiple languages) that describes your code, click Add Snippet at the bottom of the page. Otherwise, there is a place to add a new snippet at the bottom of the search results page.

If your code depends on any libraries or other snippets, there is a place at the bottom of new snippet box to add them. See this for an example of how we do external library dependencies. We want code to be instantly usable, so fill out all the dependencies if you are able, so that if someone clicks Expand Dependencies, they can drop the code anywhere and it will just work.

My favorite language isn't in the list of languages. Will you support it?

The bare minimum we need to support a language is to know its single line comment character (e.g. // in C++). So add your language to the snippets on the line comment map spec in Python and Javascript and we will support it within a day (we are working on a solution that will allow us to support it in 5 minutes). You can also add its file extension to support it in our command line tool. As we add more types of support, we'll list the specs that "understand" language features here.

I have some code from my project. Should I put on the site?

We want the code on the site to be generally reusable, so there should be a chance that somebody else could benefit from your code. A good rule of thumb is that if you need to mention the name of your project in order to describe what the function does, it shouldn't go on the site.

Why would I want to risk letting other people change my code?

It's the same philosophy as a wiki. Some people will change it for the worse, but the general good and skill of the community will tend to improve the code over time. You get the benefit of many skilled eyes looking over your code for bugs and corner cases. Besides, you always keep a local copy of any code you are using and you get a chance to review any changes before they enter your codebase. If you don't like them, you can fix it on the site for everyone to see, or just remove the tags to fork the code into your own local version independent of the site.

Just small reusable snippets? Doesn't that mean that nothing complex can ever be on the site?

No, we hope that complex code does evolve on the site. Snippets can depend on other snippets and on external libraries in each language, so complex code should be built as a composition of simpler, generally-applicable ideas, instead of as a big mass of coupled logic.

What are the licensing terms of code on the site?

Code on the site is licensed under the Creative Commons CC0 license, a public domain license. Basically you can use the code however you want, its attachment to its author is completely forfeitted. There is no warranty on the content of the site, basically don't sue anybody. By uploading code to the site you agree that you have the legal authority to licence the code under CC0.
